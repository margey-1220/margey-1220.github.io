GitHub Pages + Hugo で本サイトを作成した話

GitHub PagesとHugoを使って、本サイトを開設しました。

  • GitHub Pagesとは、GitHubアカウントを持つ人が誰でも簡単に公開サイトを作れる仕組みです。
  • Hugoとは、静的サイトジェネレータといって、文章を書いてコマンドを打つと、いい感じのサイトを生成してくれるツールです。

本サイトは、Hugoでサイトを作り、GitHub Pagesで公開しています。 まくまくさんのサイトを参考にしました。

手順は簡単です。まず、Hugoで静的サイトを生成します。

  1. Hugoをインストールする。MacでHomebrewがあるなら “brew install hugo”
  2. “hugo new site <site_name>” で新規サイトを作成する
  3. 好きなHugoのThemeをダウンロードする
  4. “hugo new /post/sample.md” で新規Webページを作成
  5. “./config.toml” を編集し、タイトルやbaseURLを変更。baseURLは公開サイトのトップページのURLにする
  6. hugo -D でサイトをビルドすると、"./public/“ディレクトリにサイトがジェネレートされる

次にジェネレートしたサイトを公開します。

  1. GitHubアカウントを作る
  2. 新規リポジトリ”<username>.github.io“を作成する
  3. 作成したリポジトリをクローンする
  4. 生成したサイト”public“ディレクトリの中身をクローンしたディレクトリに反映する
  5. git add .
  6. git commit -m "first commit"
  7. git push -u origin master

これで公開されたはずなので、https://<username>.github.io/post/にアクセスして確認します Hugoで生成されたページが見えたら成功です。

なぜHugoで作り、なぜGitHub Pagesで公開したのか

なぜか。簡単だからです。 説明した手順からわかるように、1時間もあれば個人のサイトを公開できます。（諸事情で2日かかったけどそれは諸事情があって……）

これらのデメリットはもちろんあります。

  • GitHub Pagesではサーバーサイド言語を使えないことです。PHP, Ruby, Python, などが使えないため、最近のフレームワークや高級な設計を作ることはできません。HTML+CSS+JavaScriptで構成する必要があります。
  • Hugoは他の似たツールと違って、ブラウザ上でブログの記事を編集できません。Hugoは、ビルドしてサイトを生成しなければ、公開することができません。

本サイトは、ブログとゲームの投稿をする遊び感覚で開設しました。サーバー管理の勉強やガッツリWeb開発の勉強をするならサーバー開設からやったかもしれません。しかし、そこまでの気概と時間がなかったので、楽に公開サイトを持てて、楽にサイトを作り、広く浅く楽しく自由にWeb開発したいなあと思って至った結論が、GitHub Pages + Hugoでした。

正直、軽く調べてみて自分が簡単にできそうなことをやった感はあるので、もっと良いツールや方法が絶対にありますし、目的によって変わリます。ですが、本当に楽です。こんなに楽だと思っていませんでした。記事はマークダウン形式で書けばいいだけなので、とても簡単です。慣れたら記事が大量生産できてしまいますよ。
